WHAT IS LWC? Lightning Web Components is Salesforce's modern UI framework based on native Web Component standards (Custom Elements, Shadow DOM, HTML Templates). It runs faster than Aura and uses standard JavaScript.
01 COMPONENT STRUCTURE

02 DECORATORS
@api @track @wire
@api — Public Property
import { LightningElement, api } from 'lwc';

export default class Child
  extends LightningElement {
  @api title = 'Default';
  // Parent sets via attribute
  @api greet() { // public method
    console.log('Hi!');
  }
}
@track — Reactive (Legacy)
@track obj = { name: 'SF' };
// Now auto-reactive for primitives
// @track needed only for deep objects
// in API < 46.0
@wire — Data Binding
import { wire } from 'lwc';
import { getRecord } from
  'lightning/uiRecordApi';

@wire(getRecord, {
  recordId: '$recordId',
  fields: [FIELDS]
}) wiredRecord;

// $ prefix = reactive property
All class fields are now auto-reactive. @track is only needed for deep object mutation tracking in older API versions.
03 TEMPLATE DIRECTIVES
Conditional Rendering
<!-- if:true / if:false (legacy) -->
<template if:true={isVisible}>
  <p>Shown!</p>
</template>

<!-- lwc:if (API 55+) -->
<template lwc:if={show}>Shown</template>
<template lwc:elseif={maybe}>Maybe</template>
<template lwc:else>Hidden</template>
Iteration
<template for:each={items}
         for:item="item"
         for:index="i">
  <li key={item.id}>
    {i}: {item.name}
  </li>
</template>

<!-- iterator directive -->
<template iterator:it={items}>
  <li key={it.value.id}>
    <template if:true={it.first}>FIRST</template>
    {it.value.name}
    <template if:true={it.last}>LAST</template>
  </li>
</template>

04 LIFECYCLE HOOKS
import { LightningElement } from 'lwc';

export default class Demo
  extends LightningElement {

  // 1st — property init, no DOM yet
  constructor() {
    super();
  }

  // 2nd — inserted into DOM
  connectedCallback() { }

  // 3rd — component fully rendered
  renderedCallback() { }

  // removed from DOM
  disconnectedCallback() { }

  // uncaught error in child
  errorCallback(error, stack) { }
}
HookTiming
constructor()Before render, no DOM
connectedCallback()In DOM, not rendered
renderedCallback()After every render
disconnectedCallback()Removed from DOM
errorCallback()Child error caught
Use a boolean flag in renderedCallback() to run logic only once on first render.

07 WIRE SERVICE
Wire to Property
import { wire } from 'lwc';
import getAccounts
  from '@salesforce/apex/AccCtrl.getList';

@wire(getAccounts, { type: '$accType' })
wiredAccts; // {data, error}

// In template:
// {wiredAccts.data} / {wiredAccts.error}
Wire to Function
@wire(getAccounts, { type: '$type' })
wiredHandler({ data, error }) {
  if (data) {
    this.accounts = data;
  } else if (error) {
    this.error = error;
  }
}

Across Tree (PubSub / LMS)
import { publish,
  MessageContext }
  from 'lightning/messageService';
import CHANNEL
  from '@salesforce/messageChannel/MyChannel__c';

@wire(MessageContext) ctx;

sendMsg() {
  publish(this.ctx, CHANNEL,
    { payload: 'hello' });
}

import { subscribe }
  from 'lightning/messageService';

connectedCallback() {
  this.sub = subscribe(
    this.ctx, CHANNEL, (msg) => {
    console.log(msg.payload);
  });
}

15 TESTING (JEST)
import { createElement }
  from 'lwc';
import MyComp
  from 'c/myComp';

describe('c-my-comp', () => {
  afterEach(() => {
    // Reset DOM
    while(document.body.firstChild)
      document.body.removeChild(
        document.body.firstChild);
  });

  it('renders title', () => {
    const el = createElement('c-my-comp',
      { is: MyComp });
    el.title = 'Hello';
    document.body.appendChild(el);
    const h1 = el.shadowRoot
      .querySelector('h1');
    expect(h1.textContent)
      .toBe('Hello');
  });
});

17 BEST PRACTICES
  • Use @api only for public interface
  • Prefer lwc:if over if:true (API 55+)
  • Always add key in iterations
  • Avoid DOM manipulation in constructor
  • Unsubscribe LMS in disconnectedCallback
  • Use cacheable=true for read-only wire
  • Avoid @track — not needed for primitives
  • Use getters for computed template values
  • Never mutate @api props directly
  • Use lwc:ref over querySelector when possible
Use refreshApex(wiredProperty) to reload wired data after DML.

19 QUICK CHECKLIST
Component Checklist
  • File names match folder name (camelCase)
  • Meta XML has correct apiVersion + targets
  • @api on all public props/methods
  • key set on all for:each iterations
  • lwc:if used for conditional (API 55+)
  • Error + loading states handled in wire
  • Events: detail, bubbles, composed set
  • LMS subscription cleaned up on disconnect
  • refreshApex called after DML mutations
  • Jest tests cover happy + error paths
